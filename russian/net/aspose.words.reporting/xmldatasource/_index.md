### Примечания

Для доступа к данным соответствующего файла или потока при создании report, передайте экземпляр этого класса как источник данных в одну из перегрузок[`ReportingEngine`](../reportingengine).BuildReport.

В шаблонных документах, если элемент XML верхнего уровня содержит только список элементов одного type, an[`XmlDataSource`](../xmldatasource)instance следует обрабатывать так же, как если бы он был aDataTable экземпляр. В противном случае экземпляр[`XmlDataSource`](../xmldatasource)следует обрабатывать так же, как если бы он был aDataRow экземпляр. Дополнительные сведения см. в справочнике по синтаксису шаблона (https://docs.aspose.com/display/wordsnet/Template+Syntax).

Когда определение схемы XML передается в конструктор этого класса, типы данных значений простого XML элементы и атрибуты определяются в соответствии со схемой. Таким образом, в документах-шаблонах вы можете работать с типизированными значениями , а не только со строками.

Когда определение схемы XML не передается в конструктор этого класса, типы данных значений простых Элементы XML и атрибуты определяются автоматически по их строковым представлениям. Таким образом, в шаблонных документах вы можете работать с типизированными значениями и в этом случае. Движок способен автоматически распознавать значения следующих типов:

* Nullable
* Nullable
* Nullable
* Nullable
* String

Обратите внимание, что для работы автоматического распознавания типов данных строковые представления значений простых элементов XML и атрибутов должны формироваться с использованием инвариантных настроек культуры.

Чтобы переопределить поведение загрузки данных XML по умолчанию, инициализируйте и передайте[`XmlDataLoadOptions`](../xmldataloadoptions) экземпляр конструктору этого класса.
