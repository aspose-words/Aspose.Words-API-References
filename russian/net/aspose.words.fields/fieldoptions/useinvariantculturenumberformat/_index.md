## FieldOptions.UseInvariantCultureNumberFormat property

Получает или задает значение, указывающее, анализируется ли числовой формат с использованием инвариантного языка и региональных параметров

```csharp
public bool UseInvariantCultureNumberFormat { get; set; }
```

### Примечания

Когда для этого свойства установлено значение **true** , числовой формат берется из инварианта культура.

Когда для этого свойства установлено значение **false** , числовой формат берется из культуры текущего потока.

Значение по умолчанию: **false** .

### Примеры

Показывает, как форматировать числа в соответствии с инвариантной культурой.

```csharp
Document doc = new Document();
DocumentBuilder builder = new DocumentBuilder(doc);

Thread.CurrentThread.CurrentCulture = new CultureInfo("de-DE");
Field field = builder.InsertField(" = 1234567,89 \\# $#,###,###.##");
field.Update();

// Иногда поля могут неправильно форматировать свои числа в некоторых культурах. 
Assert.IsFalse(doc.FieldOptions.UseInvariantCultureNumberFormat);
Assert.AreEqual("$1234567,89 .     ", field.Result);

 // Чтобы исправить это, мы могли бы изменить культуру для всего потока.
 // Другой способ исправить это - установить этот флаг, 
 // что заставляет все поля использовать инвариантную культуру при форматировании чисел.
 // Этот способ позволяет избежать изменения культуры для всего потока.
doc.FieldOptions.UseInvariantCultureNumberFormat = true;
field.Update();
Assert.AreEqual("$1.234.567,89", field.Result);
```
