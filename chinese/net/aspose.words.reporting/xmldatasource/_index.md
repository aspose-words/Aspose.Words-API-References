### 评论

访问相应文件或流的数据，同时生成一个报告，将此类的实例作为 数据源传递给[`ReportingEngine`](../reportingengine).BuildReport 重载之一。

在模板文档中，如果顶级 XML 元素仅包含相同元素的列表type, an[`XmlDataSource`](../xmldatasource)实例应该以与 a:::相同的方式处理R5:T:System.Data.DataTable::: 实例。否则，[`XmlDataSource`](../xmldatasource)实例应以与 aDataRow 实例。有关详细信息，请参阅模板语法参考 (https://docs.aspose.com/display/wordsnet/Template+Syntax)。

当 XML Schema Definition 传递给此类的构造函数时，简单 XML 值的数据类型元素 和属性是根据模式确定的。因此，在模板文档中，您可以使用类型值 而不仅仅是字符串。

当 XML Schema Definition 没有传递给此类的构造函数时，简单的值的数据类型XML 元素 和属性是根据它们的字符串表示自动确定的。因此，在模板文档中，您也可以使用 在这种情况下使用键入的值。引擎能够自动识别以下类型的值:

* Nullable
* Nullable
* Nullable
* Nullable
* String

请注意，为了使数据类型的自动识别起作用，简单 XML 元素 和属性的值的字符串表示应使用不变的文化设置形成。

要覆盖 XML 数据加载的默认行为，初始化并传递[`XmlDataLoadOptions`](../xmldataloadoptions) 实例到此类的构造函数。
